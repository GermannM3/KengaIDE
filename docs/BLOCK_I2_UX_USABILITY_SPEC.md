# I2-UX-USABILITY-TEST — Сценарии и Friction Points

> Валидация UX перед I3. Сценарии, friction points, критерии готовности.

---

## 0. Цель

Проверить, что I2-UX-POLISH **работает в реальных сценариях**:
пользователь достигает цели без когнитивного шума и тупиков.

---

## 1. Ключевые сценарии

### S1. Первый запуск (новый пользователь)

| Шаг | Ожидание | Критерий |
|-----|----------|----------|
| Запуск | Welcome, понятные действия | ≤ 3 клика до «Новый проект» или «Открыть папку» |
| Выбор модели | Рекомендация, не перегруз | DeepSeek для coding — явно рекомендован |
| Первый проект | Создан и открыт | Дерево файлов видно, редактор готов |

### S2. Запуск агента

| Шаг | Ожидание | Критерий |
|-----|----------|----------|
| Run Agent | Диалог ввода задачи | Один способ: меню или palette |
| Агент работает | Timeline виден, прогресс понятен | Stop всегда доступен |
| Завершение | Результат + Retry/Rollback | Пользователь понимает, что произошло |

### S3. Рефакторинг кода

| Шаг | Ожидание | Критерий |
|-----|----------|----------|
| Открыт файл | Editor в фокусе | — |
| Refactor | Диалог или inline | Патч виден до apply |
| Откат | Rollback за 1 клик | Понятно, что откатится |

### S4. Policy / Enterprise

| Шаг | Ожидание | Критерий |
|-----|----------|----------|
| Ограничение активно | UI отражает | Disabled пункты с пояснением |
| Audit | Сессия доступна | Кнопка «Аудит», папка открывается |
| Status Bar | Policy виден | «Enterprise · Locked» или аналог |

### S5. Смена контекста

| Шаг | Ожидание | Критерий |
|-----|----------|----------|
| Открыть другой проект | Без потери состояния | Или явное предупреждение |
| Сменить модель | Модель в status bar | Текущая модель всегда видна |
| Split view | Файлы в двух панелях | Ctrl+Click работает |

---

## 2. Friction Points (чеклист)

Проверить и устранить:

- [ ] «Где я?» — project path всегда виден
- [ ] «Что делает AI?» — timeline читаемый, не raw log
- [ ] «Как остановить?» — Stop очевиден
- [ ] «Как откатить?» — Rollback с preview
- [ ] «Почему недоступно?» — disabled с tooltip
- [ ] «Куда сохраняется?» — путь в Save As
- [ ] «Какой режим AI?» — Agent/Chat в status bar
- [ ] «Записывается ли?» — Audit индикатор

---

## 3. Метрики (опционально)

| Метрика | Целевое |
|---------|---------|
| Time to first project | ≤ 2 мин |
| Time to first agent run | ≤ 1 мин (при готовой модели) |
| Clicks to Stop | 1 |
| Clicks to Rollback | 2 (подтверждение) |

---

## 4. Definition of Done

I2-UX-USABILITY готов, если:

- Все 5 сценариев проходят без тупиков
- Friction points закрыты или задокументированы
- Нет «магических» состояний (пользователь всегда понимает контекст)
