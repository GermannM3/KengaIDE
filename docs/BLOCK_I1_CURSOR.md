# BLOCK I1-CURSOR — AI-инструментарий разработки

## Цель

Обеспечить **контролируемую, воспроизводимую и безопасную** разработку с Cursor / AI IDE без превращения проекта в «магическую кашу».

## Роль Cursor

- AI-помощник разработчика
- ускоритель рефакторинга
- инструмент навигации по коду
- генератор шаблонов (boilerplate), не логики

**Запрещено:** генерировать бизнес-логику без ревью, менять архитектуру без ADR, автокоммиты без человека.

## Структура .ai/

```
.ai/
├── system.md        # кто мы, что строим, философия
├── architecture.md  # слои, зависимости, защищённые зоны
├── rules.md         # что AI можно / нельзя
├── prompts/
│   ├── refactor.md
│   ├── testgen.md
│   └── docs.md
└── README.md
```

## Защищённые зоны

Изменения только с явным флагом:

- `crates/backend_core/src/audit.rs`
- `crates/backend_core/src/command_router.rs`
- `crates/ai_runtime/src/orchestration.rs`
- `crates/ai_runtime/src/agent.rs`
- `crates/ai_providers/src/traits.rs`

## Контроль качества

1. Diff-review
2. Читаемость, стиль, отсутствие скрытой логики
3. Чеклист: нет магических чисел, hardcode путей, побочных эффектов

## Безопасность

- Не отправлять ключи, лицензии, installer-скрипты
- `.env` в `.gitignore`
- Чувствительный код: `// AI-NO-TOUCH`

## Cursor + Installer

Cursor **не генерирует** финальный установщик. Может: шаблон, описание flow, UI-экраны. Сборка, sandbox, изоляция — всегда руками.

## Метрика пользы

AI полезен, если: ускоряет ≥30%, снижает баги, не усложняет onboarding. Иначе — отключается.
